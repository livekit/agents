memray captures

captures from one or more memray run --follow-fork sessions, grouped by worker

processsizecapturedfile

Each group is one worker invocation; rows underneath are its forked child processes (jobs and the inference process). Click into any flamegraph and switch to the peak view — for a flat, non-growing process that peak is essentially the whole resident footprint. Tall native frames (torch, onnxruntime, model weights) are usually the suspects.
